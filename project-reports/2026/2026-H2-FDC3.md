# Project Overview

[FDC3](https://fdc3.finos.org) is an open standard for application interoperability on the financial desktop. It defines a shared language based on **context** objects (shared data), **intents** (actions), and a broker (the **Desktop Agent**) for communication. The standard currently consists of five parts: the API, App Directory, Intents, Context Data, and Agent Bridging specifications. FDC3 is in production at major banks, asset managers and vendors worldwide.

# Current Status

This reporting period has been dominated by work on **FDC3 3.0**, which is focused on four themes: Observability, Security, Cross-Firm Interoperability, and Cross-Technology Support.

Key accomplishments:

- **FDC3 3.0 Alpha released** — `@finos/fdc3@3.0.0-alpha.2` published to npm with a fully working reference implementation for FDC3 for Web Browsers.
- **Security & Identity** — the headline feature of 3.0. Delivered signed contexts (JWS), encrypted channels (JWE/AES-GCM), symmetric key exchange, user identity (`GetUser` intent), and anti-replay protection. A new `@finos/fdc3-security` reference library ships with samples and example applications. The security model is end-to-end (DA is an untrusted intermediary) with a trusted-backend contract for cryptographic operations.
- **Metadata & Observability** — `ContextMetadata` is now a required feature of the API with `source`, `timestamp`, `traceId`, `signature`, `antiReplay`, and extensible `custom` fields. Apps can provide metadata when broadcasting or raising intents via `AppProvidableContextMetadata`.
- **Language Interoperability** — FDC3 3.0 will ship official bindings for TypeScript, .NET/C#, Go (new), and Java (new). A proposed WebSocket Connection Protocol (WSCP) enables any language with WebSocket support to implement an FDC3 adaptor.
- **API Improvements** — `clearContext()` on channels, `fdc3.close()`, `addIntentListenerWithContext()`, intent listener conflict detection, metadata API functions, destructuring support, fully-qualified appIds, and removal of deprecated functions from 2.0–2.2.
- **npm v2.2.2 and v2.2.3 released** (April 2026) — bug fixes, conformance test improvements, dynamic intent listener support, and the conformance project ported into the monorepo.
- **CI/CD hardening** — all GitHub Actions pinned to immutable commit SHAs, StepSecurity harden-runner added, Dependabot configured with cooldown periods, `min-release-age = 7` added to `.npmrc`.
- **Presented at OSFF London 2026** — "FDC3: What's New in 3.0?" covering security, metadata, language bindings and a live FDC3 Sail 3.0 demo.

# Challenges & Blockers

- **Contribution concentration:** The lead maintainer accounts for a significant majority of commits (~50% in H2). While the maintainer group is diverse (9 maintainers across 6 organisations), active commit contribution remains concentrated. Broadening hands-on contribution is a priority.
- **3.0 scope breadth:** FDC3 3.0 is ambitious — security, metadata, 4 language bindings, API changes, WSCP, and conformance tests. Coordinating delivery across working groups while maintaining quality requires sustained effort.
- **Adoption of new security features:** The security model requires a trusted backend and JWKS infrastructure. Adoption will depend on clear documentation, samples, and platform provider support.
- **Commit velocity decline in mid-2025:** Activity dipped in mid-2025 (between the 2.2 standard release and the start of 3.0 development) but recovered strongly from late 2025 onward as 3.0 work ramped up.

# Roadmap & Goals for Next 6 Months

- **FDC3 3.0 Standard release** — complete the remaining 3.0 scope items, achieve Standards Working Group vote approval, and publish the final 3.0 standard.
- **Go and Java language bindings** — finalise and release official bindings alongside the 3.0 standard.
- **WebSocket Connection Protocol (WSCP)** — standardise the protocol enabling non-browser language bindings to connect to Desktop Agents.
- **Conformance test suite for 3.0** — extend the conformance framework to cover metadata, security, `clearContext`, `fdc3.close()`, and other new features.
- **FDC3 Sail 3.0** — support the FINOS FDC3 Sail project in delivering a production-ready Desktop Agent implementing FDC3 3.0.
- **Grow contributor base** — onboard new maintainers and editors, curate "good first issue" tags, and leverage conference presence (OSFF) for recruitment.
- **Documentation & training** — update fdc3.finos.org for 3.0, including security guides and migration documentation from 2.2.

# TOC Support Needed

- **Cross-project collaboration** — FDC3 Sail is a separate FINOS project that implements the FDC3 standard. Coordination between the two projects' release timelines would benefit from TOC visibility.
- **Adoption & visibility** — amplifying FDC3 3.0's security and multi-language capabilities to the broader FINOS community and financial services industry.
- **Contributor diversity** — guidance or programmes to help reduce contribution concentration and attract new organisational contributors.

# Additional Information

- FDC3 is an **npm workspaces monorepo** with 10+ packages. Build order matters: `fdc3-schema` → `fdc3-standard` → `fdc3-agent-proxy` / `fdc3-web-impl` → `fdc3-get-agent`.
- Tests use Cucumber/Gherkin with `quickpickle` + `vitest`. The conformance test suite validates Desktop Agent compliance.
- Documentation: https://fdc3.finos.org · Free training: https://www.edx.org/course/fdc3-interoperability-for-the-financial-desktop
- The project maintains active working groups: Standards WG (monthly), Identity & Security WG (monthly), Use Cases & Workflows (monthly), Web Browsers & Bridging (monthly), and a General Meeting (every 6 months).
